# /merge-gate — 머지 게이트 (장치 5)

> **한 줄 요지:** 여러 팀이 main에서 브랜치를 따 병렬로 일할 때, **오래된 base에서 딴 브랜치가 최신화 없이 머지되어 남의 최신 작업을 조용히 되돌리는 롤백**을 막는다. 머지 직전에 다섯 가지를 확인해, 안전하면 통과·직렬 머지하고 위험이면 결정 큐로 올린다. KimOrchestrator의 통합 권한을 구현한 장치다.

## 0. 왜 이 게이트가 있나 — 롤백의 기전

main이 A일 때 브랜치 X·Y를 둘 다 A에서 딴다. Y가 먼저 머지돼 main이 B가 된다. X는 여전히 옛 A 기준으로 `STATUS.md`·디자인 토큰·`sitemap-erd.html`·마이그레이션 같은 **공유 파일**을 건드린 채 머지되고, 그 충돌을 X의 옛 내용 쪽으로 풀면 **Y의 최신 작업이 조용히 사라진다.** 이 게이트는 **공유 표면 + 오래된 base + 직렬화 안 된 머지** 세 조건이 겹치는 지점을 머지 직전에 끊는다.

## 1. 두 겹 — 설정이 대부분, 게이트는 판단만

- **가드레일 A(GitHub 설정):** main 브랜치 보호에 "머지 전 최신화 강제 + 머지 큐 + CI 필수"가 켜져 있으면, 오래된 base 머지와 동시 머지 레이스는 GitHub가 원천 차단한다. 이 게이트는 그 위에서 **설정이 못 보는 것**만 본다.
- **게이트 B(이 커맨드):** "이 머지가 남의 것을 지우나 / 열린 다른 PR과 같은 계약을 건드리나" — 사람 판단이 필요한 것만 판정해 올린다.

가드레일 A가 아직 안 켜져 있으면, 게이트가 1단계(최신화)를 직접 확인해 보류시킨다.

## 2. 머지 직전 5단계

1. **최신화 확인** — 대상 PR이 최신 main 기준인가(behind면 보류). `pull_request_read`의 mergeable_state, 또는 `git merge-base --is-ancestor origin/main <PR head>`로 확인. behind면 **보류** — "최신 main으로 갱신 후 CI 재통과" 지시.
2. **pr-gate 재실행** — `/pr-gate <PR>`을 머지 직전 상태로 다시 돌려 척추·문서 매트릭스·타 파트 영향을 확인.
3. **계약 충돌 교차 탐지** — 계약 레지스트리로, **지금 열린 다른 PR 중 같은 공유물(파일·표·토큰·흐름)을 건드리는 게 있나** 대조한다. 각 열린 PR의 변경 파일(`pull_request_read` get_files)을 공유물에 매핑하고, 둘 이상이 같은 계약에 걸리면 **충돌 후보** — 소비자 파트와 함께 결정 큐로 올린다.
4. **되돌림 사전 시뮬레이션** — 임시 워크트리에서 `git merge --no-commit --no-ff`로 병합을 시늉만 낸 뒤 base와 diff한다. 병합 결과가 **main의 최근 커밋이 추가한 줄을 삭제(net 제거)** 하면 되돌림으로 판정해 **차단**. 순수 추가·비충돌은 통과. (충돌 마커가 나면 그 자체가 최신화 필요 신호 → 보류.)
5. **머지 큐로 하나씩** — 통과분만 머지 큐(또는 순차 머지)에 넣는다. 여러 개면 계약을 안 겹치는 것부터, 겹치면 한 번에 하나씩.

## 3. 판정

- **통과** — 최신·게이트 통과·계약 미충돌·되돌림 없음 → 머지 큐로.
- **보류** — behind이거나 CI 미통과 → 최신화·재검증 요청(사람/팀이 브랜치 갱신).
- **차단** — 되돌림 위험 또는 계약 충돌 → `/sysreport` 결정 큐로 올려 KimOrchestrator가 대표에게.

## 4. 한계 (정직하게)

- **텍스트 충돌만** 잡는다. 서로 다른 파일을 고쳤지만 의미상 충돌(한쪽이 지운 컬럼을 다른 쪽이 참조 등)은 이 dry-run으로 안 잡힌다 — 그건 2단계 pr-gate의 척추·계약 검사와 e2e가 겹쳐 잡아야 한다.
- **가드레일 A(브랜치 보호)는 사람이 설정에서 켜야** 하고, 게이트가 대신 켜지 못한다.
- **계약 레지스트리가 최신**이어야 3단계 교차 탐지가 정확하다.

## 5. 절제

- PR 하나짜리 사소한 머지(오타·문서 한 줄, 공유물 무관)엔 5단계를 다 돌리지 않고 최신화만 확인해 통과. 판돈에 맞춘다.
- 위험 없는 정상 머지는 조용히 통과시키고 현황판에 기록만 — 대표에게 올리는 것은 보류·차단뿐.
